# §1.6 Agent Loop / LLM Integration

Implementation notes for the agent loop that connects an LLM (e.g. Ollama/qwen3) to the security runtime.

---

## Troubleshooting

### `ModuleNotFoundError: No module named 'src'`

When you run:

```bash
agent-loop --policy examples/policies/Policy.yaml --prompt "run git status" --model qwen3
```

you may see:

```
ModuleNotFoundError: No module named 'src'
```

**Cause:** The `agent-loop` entry point runs in a context where the project root is not on `sys.path`, so `import src` (used in `src.runtime.agent_loop`) fails.

**Fix:**

1. **Install the package in editable mode from the repo root** (recommended):
   ```bash
   cd M:\format-1-28-2026\ECE50874\Security-Constrained-Agent-Runtime
   pip install -e .
   ```
   Then run `agent-loop` from the same machine; the installed package will resolve `src` correctly.

2. **Or run from the repo root with PYTHONPATH set:**
   ```bash
   cd M:\format-1-28-2026\ECE50874\Security-Constrained-Agent-Runtime
   set PYTHONPATH=%CD%
   agent-loop --policy examples/policies/Policy.yaml --prompt "run git status" --model qwen3
   ```
   On PowerShell:
   ```powershell
   $env:PYTHONPATH = (Get-Location).Path
   agent-loop --policy examples/policies/Policy.yaml --prompt "run git status" --model qwen3
   ```

3. **Or run the module directly from the repo root:**
   ```bash
   cd M:\format-1-28-2026\ECE50874\Security-Constrained-Agent-Runtime
   python -m src.runtime.agent_loop --policy examples/policies/Policy.yaml --prompt "run git status" --model qwen3
   ```

Always run from the **repository root** (where `setup.py` and `src/` live) when using options 2 or 3.
